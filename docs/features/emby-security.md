# 开服配置

对应前端侧边栏 **开服 → 开服配置**，集中管理注册策略、定时任务与安全防护，所有配置通过顶部「保存全局配置」按钮统一保存。

## 页面结构

页面分为三大卡片区域：**定时任务**、**注册配置**（按实例分组）和**安全防护**。

---

## 定时任务

控制到期扫描和用户同步的自动执行策略。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **到期扫描** | `0 3 * * *` | 自动禁用到期用户并通知管理员。可开关控制是否启用 |
| **Emby 用户同步** | `0 */3 * * *` | 定期从 Emby 同步用户列表，留空可关闭 |

::: tip 到期扫描与自动续费联动
如果在 [实例管理](emby-instances.md) 中开启了积分自动续费，到期扫描会先尝试用积分续费，积分不足时才禁用用户。
:::

## 注册配置

注册配置按实例分组展示，每个实例独立控制注册策略。

### 每个实例的配置项

| 配置项 | 说明 |
|--------|------|
| **开放注册** | 开关。开启后允许用户通过注册页面或 Bot 自助注册该实例 |
| **邀请码** | 开关。开启后注册时必须提供有效邀请码 |
| **默认网盘模式** | 仅自备 / 仅共享 / 共享+自备（可选默认自备或默认共享） |
| **默认时长档位** | 天卡 / 周卡 / 月卡 / 年卡 / 白名单。启用邀请码时以邀请码时长为准 |

### 配置逻辑

::::: steps

1. **开放注册**

   打开开关前，系统会校验该实例是否已设置模板用户（在 Emby 助手中配置）。模板用户用于复制权限策略给新注册用户

2. **邀请码控制**

   开启邀请码后，「默认时长档位」变为灰色，因为此时以邀请码中设定的时长为准

3. **多实例注册**

   用户在注册页面可通过下拉菜单选择要注册的实例，切换实例时自动清空邀请码输入框

:::::

::: caution 开放注册后，任何人均可注册。建议同时开启「邀请码」来控制注册范围。
:::

## 安全防护

防爆破保护与网关流控参数，所有参数**实时生效**，无需重启服务。

### 总开关

| 配置项 | 说明 |
|--------|------|
| **启用安全防护** | 总开关，关闭后所有防爆破限流停用 |

### 登录防爆破

同一 IP 在窗口时间内登录失败超过阈值后，该 IP 将被封禁指定时长。同时按用户名维度也有独立限流，登录成功后自动解除。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **最大尝试次数** | 5 | 超限后触发封禁 |
| **窗口时长（秒）** | 60 | 失败计数时间窗口 |
| **封禁时长（秒）** | 900 | 触发后拒绝登录时长（默认 15 分钟） |

### 注册防爆破

防止同一 IP 短时间内批量注册账号。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **最大注册次数** | 3 | 同一 IP 在窗口内允许的最大注册次数 |
| **窗口时长（秒）** | 60 | 注册计数时间窗口，超限后冷却时长等于窗口时长 |

### 密码重置防爆破

防止攻击者大量请求验证码骚扰已绑定 Telegram 的用户。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **最大重置次数** | 3 | 发送验证码和确认重置的最大次数 |
| **窗口时长（秒）** | 60 | 重置请求计数时间窗口 |

### 设备流控

按设备标识追踪每用户在线设备数，超限拒绝登录和播放。详见 [活跃设备](emby-sessions.md)。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **默认设备数上限** | 1 | 新用户创建/同步时的默认值，填 0 不限制 |

::: info 设备流控细节
- 管理员账号自动跳过设备流控
- 仅限制登录和播放，浏览媒体库不受影响
- 设备空闲 5 分钟后，新设备可自动替换旧设备
- 可在 [Emby 用户](emby-user-mgmt.md) 中为每个用户单独覆盖设备上限
:::
