# 常见问题 (FAQ)

## 部署相关

### Q: 支持哪些系统部署？

A: ResourceTool 通过 Docker 部署，支持所有运行 Docker 的系统：
- Linux（Ubuntu、Debian、CentOS 等）
- NAS 设备（群晖、威联通等）
- macOS
- Windows（通过 Docker Desktop 或 WSL2）

### Q: 端口被占用怎么办？

A: 检查端口占用：
```bash
ss -tlnp | grep 29999
```
如需更改端口，可以修改 docker-compose.yml 中的环境变量 `RESOURCE_WEB_PORT`。

### Q: 容器无法启动？

A: 查看容器日志排查原因：
```bash
docker logs resource-tool
```

常见原因：
- 授权码未配置或已过期
- 端口被占用
- 数据目录权限不足

### Q: 如何查看当前版本？

A: 有以下几种方式：
1. 面板右下角显示版本号
2. Telegram Bot 发送 `/version`
3. 查看容器内 `/VERSION` 文件

## 115 网盘相关

### Q: Cookie 多久过期？

A: 115 Cookie 的有效期不固定，通常在 7-30 天之间。系统会自动检测并提醒。建议定期更新 Cookie。

### Q: 115 出现 405 风控怎么办？

A: 405 是 115 的接口限流机制，通常原因：
- 短时间内请求过多
- Cookie 异常

处理方式：
1. 系统会自动触发熔断器，暂停请求
2. 等待一段时间后自动恢复
3. 如果持续出现，检查 Cookie 是否正常
4. 考虑配置多个 115 账号分流

### Q: 转存失败常见原因？

A: 
- **Cookie 过期** — 更新 Cookie
- **目标目录不存在** — 检查转存目录配置
- **分享链接已失效** — 资源被取消分享
- **存储空间不足** — 检查 115 网盘空间

## Telegram 相关

### Q: Bot 无法接收消息？

A: 检查以下事项：
1. Bot Token 是否正确
2. 是否配置了代理（Telegram 需要科学上网）
3. 查看日志是否有连接错误

### Q: Session 登录失败？

A: 常见原因：
1. 网络不通（需要代理）
2. 二步验证密码错误
3. Telegram 账号被限制

### Q: 频道监控不生效？

A: 检查：
1. Session 是否在线
2. 监控规则是否已启用
3. 源频道是否已加入（使用 Session 的账号需要是频道成员）
4. 关键词过滤是否设置正确

## Emby 302 相关

### Q: 播放时转圈/缓冲？

A: 可能原因：
1. 115 直链获取慢 — 开启预缓存
2. 115 CDN 速度慢 — 检查网络
3. 风控触发 — 查看日志是否有 405 错误
4. CD2 降级中 — 检查 CD2 连接状态

### Q: 路径映射不匹配？

A: 检查：
1. STRM 文件中的路径格式
2. 路径映射规则是否与 STRM 路径前缀一致
3. Cloud STRM 中 `False` 后面是否有两个斜杠 `//`

### Q: 秒传失败？

A: 可能原因：
1. 目标 115 账号 Cookie 过期
2. 目标目录不存在
3. 文件已存在于目标账号

## 用户管理相关

### Q: 忘记管理员密码？

A: 默认密码为 `admin`。如果修改后忘记，需要通过数据库工具重置。

### Q: 自备 Cookie 用户如何绑定？

A: 
1. 管理员在 115 配置中添加用户的 Cookie
2. 在用户管理中将用户模式设为 `self_pan`
3. 关联对应的 115 配置

### Q: 用户到期后如何续期？

A: 在用户管理中编辑用户，修改到期时间即可。也可以通过激活码让用户自助续期。
