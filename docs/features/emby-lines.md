# 访问线路

对应前端侧边栏 **开服 → 访问线路**，为每个 Emby 实例配置多条访问线路，用户可通过 Bot `/status` 命令或用户面板查看和切换线路。

::: important 核心用途
当 Emby 有多条接入线路（如不同地区 CDN、不同端口）时，通过访问线路管理可以让用户自行选择最优线路，同时支持隐藏特定线路仅对白名单/永久用户开放。
:::

## 页面结构

顶部有实例选择下拉、「探测」「添加」「保存」按钮。下方为线路列表，每条线路为一行内联编辑的卡片。

---

## 线路配置

每条线路包含以下配置项，所有字段直接在列表行内编辑：

| 配置项 | 说明 |
|--------|------|
| **线路名称** | 自定义名称（如「深圳电信」「香港 CN2」） |
| **协议** | 下拉选择 `https` 或 `http`，切换协议时端口自动适配（443↔80） |
| **域名/IP** | Emby 服务器的域名或 IP 地址 |
| **端口** | 访问端口，使用默认端口（https=443, http=80）时 URL 中不显示端口号 |
| **启用** | 勾选框，取消勾选后该线路不对用户展示 |
| **隐藏** | 勾选框，隐藏线路不对普通用户展示 |
| **白名单可见** | 仅在勾选「隐藏」后可设置，勾选后白名单和永久用户仍可看到此线路 |

::: tip URL 字段拆分
线路 URL 被拆分为协议、域名、端口三个独立字段，避免手动拼接 URL 出错。系统自动合成完整 URL（如 `https://emby.example.com:8920`），默认端口会自动省略。
:::

### 线路可见性规则

| 用户类型 | 普通线路 | 隐藏线路 | 隐藏+白名单可见 |
|---------|---------|---------|----------------|
| 普通用户 | ✅ 可见 | ❌ 不可见 | ❌ 不可见 |
| 永久用户 | ✅ 可见 | ❌ 不可见 | ✅ 可见 |
| 白名单用户 | ✅ 可见 | ❌ 不可见 | ✅ 可见 |
| 管理员 | ✅ 可见 | ✅ 可见 | ✅ 可见 |

::: info 用户面板线路展示
用户面板中的线路列表会智能折叠：当多条线路使用相同协议和端口时，仅展示域名差异部分，减少视觉干扰。
:::

## 使用方法

::::: steps

1. **选择实例**

   顶部下拉选择要配置线路的 Emby 实例

2. **添加线路**

   点击「添加」按钮新增一行，填写线路名称、协议、域名和端口

3. **设置可见性**

   根据需要勾选「启用」「隐藏」「白名单可见」

4. **调整排序**

   按住线路左侧的拖拽手柄上下拖动调整顺序，顺序决定用户看到的排列

5. **保存**

   点击「保存」按钮提交所有更改

:::::

## 线路探测

点击顶部「探测」按钮，系统自动测试所有线路的可用性和响应延迟：

- <Badge type="tip" text="在线" /> — 显示延迟毫秒数（如 `45ms`）
- <Badge type="danger" text="离线" /> — 线路不可达

::: tip 探测结果仅供参考，实际用户体验还受地理位置和网络环境影响。
:::
