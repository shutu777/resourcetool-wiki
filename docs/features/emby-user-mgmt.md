# Emby 用户

对应前端侧边栏 **开服 → Emby 用户**，是 Emby 开服的核心管理页面，管理所有 Emby 实例下的用户。

## 页面结构

顶部标题栏包含以下操作按钮：

:::: card-grid
::: card title="创建用户" icon="material-symbols:person-add"
手动创建新 Emby 用户，设置实例、密码、网盘模式和到期时间
:::
::: card title="从 Emby 导入" icon="material-symbols:download"
从 Emby Server 批量导入已有用户到 RT 数据库
:::
::: card title="同步到 Emby" icon="material-symbols:sync"
将 RT 用户批量同步到 Emby Server（创建/更新状态）
:::
::::

下方为搜索栏（按用户名搜索）+ Emby 实例筛选下拉 + 状态筛选下拉 + 网盘模式筛选下拉。

### 用户列表表格

| 列名 | 说明 |
|------|------|
| **用户名** | Emby 用户名 |
| **Emby 实例** | 所属 Emby 302 实例名称 |
| **网盘模式** | 「共享」或「自备」标签 |
| **状态** | active（绿色）/ disabled（红色）/ expired（琥珀色） |
| **白名单** | <Badge type="tip" text="白名单" /> 标记，白名单用户不受到期扫描影响 |
| **到期时间** | 到期日期，永久用户显示「永久」标签 |
| **积分** | 用户当前积分余额（签到模式下显示） |
| **TG 绑定** | 已绑定的 Telegram 用户名或 ID |
| **设备数** | 当前设备数 / 设备上限 |
| **最后签到** | 最近一次签到时间（签到模式下显示） |
| **最后播放** | 最近一次播放时间 |
| **操作** | 编辑 / 同步到 Emby / 删除 |

::: info 白名单 vs 永久
- **永久用户** — 没有到期时间，但不具备白名单特权
- **白名单用户** — 具备特权：不受到期扫描影响、可访问隐藏线路和受管控媒体库

两者独立设置，可以是「永久 + 白名单」也可以是「有到期时间 + 非白名单」。
:::

## 创建用户

点击「创建用户」按钮打开弹窗：

| 配置项 | 说明 |
|--------|------|
| **Emby 实例** | 选择用户所属的 Emby 302 实例 |
| **用户名** | Emby 用户名 |
| **密码** | Emby 密码 |
| **网盘模式** | 共享 / 自备 |
| **到期时间** | 日期选择器，或勾选「永久」 |

::: tip 新建用户自动应用实例配置
创建用户时，系统自动应用该实例的默认设备上限和媒体库屏蔽规则，无需手动设置。
:::

---

## 编辑用户

点击用户行的编辑按钮打开弹窗，除创建时的字段外，还可编辑：

| 配置项 | 说明 |
|--------|------|
| **状态** | active / disabled |
| **白名单** | 开关，标记为白名单用户 |
| **积分** | 手动调整用户积分余额 |
| **设备上限** | 最大同时播放设备数，填 0 不限制 |
| **TG 用户 ID** | 绑定的 Telegram 用户 ID |
| **115 配置** | 自备模式下绑定的 115 账号配置 |
| **续期** | 快捷续期按钮（+7天 / +30天 / +90天 / +365天 / 永久） |

---

## 批量操作

表格支持多选，选中用户后顶部出现批量操作工具栏：

| 操作 | 说明 |
|------|------|
| **批量设备数** | 下拉选择设备上限：1/2/3/5/10 台、不限制、封禁 |
| **批量同步** | 将选中用户同步到 Emby |
| **批量删除** | 删除选中用户 |

::: tip 批量设备数修改
常见场景：活动期间临时放宽设备限制，活动结束后批量恢复。选中所有用户 → 批量设备数 → 选择目标值，一键完成。
:::

## 从 Emby 导入

::::: steps

1. **点击「从 Emby 导入」**

   弹出导入弹窗，选择 Emby 实例

2. **系统自动对比**

   拉取 Emby Server 上的用户列表，与 RT 数据库对比

3. **查看导入结果**

   | 情况 | 处理 |
   |------|------|
   | 用户不存在 | 自动创建，应用实例的同步导入默认值（时长和白名单） |
   | 同名同实例 | 跳过，标记为「已存在」 |

:::::

::: info 导入默认值
导入用户的默认时长和白名单状态由 [实例管理](emby-instances.md) 中的「同步导入默认值」配置决定。
:::

## 同步到 Emby

::: info 同步策略
- **新用户** — 在 Emby 上创建账号并设置权限
- **已有用户** — 仅同步启用/禁用状态，**不覆盖**已有权限设置（包括媒体库访问权限、字幕偏好等）

这是为了避免管理员在 Emby 后台手动调整过的个性化权限被覆盖。
:::
