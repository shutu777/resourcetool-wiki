# 设备流控

设备流控系统用于限制每个 Emby 用户同时使用的客户端数量，防止账号共享滥用。

## 功能概述

| 功能 | 说明 |
|------|------|
| **设备数限制** | 限制每用户同时在线的客户端数量 |
| **双层超时** | 空闲 5 分钟可被替换，30 分钟完全移除 |
| **平滑过渡** | 不会立即中断现有播放 |
| **活跃设备管理** | 管理员实时查看和踢出设备 |

## 流控范围

流控**仅作用于**以下三个关键环节：

- 登录认证
- 播放信息获取
- 视频播放

以下操作**不受影响**：

- 浏览媒体库
- 加载海报图片
- 搜索内容
- 查看详情页

## 双层超时机制

| 层级 | 超时时间 | 效果 |
|------|---------|------|
| **空闲超时** | 5 分钟 | 设备可被新设备替换 |
| **完全过期** | 30 分钟 | 设备从在线列表中移除 |

当用户关闭一个客户端后，5 分钟内该设备位置就可以被新设备使用，无需等待 30 分钟。

## 平滑过渡

对于更新前已在多个设备登录的用户：

- 系统**不会立即中断**正在播放的设备
- 下次播放时自动清理最久未使用的多余设备
- 逐步收敛到设备上限

## 设备数配置

### 全局默认

在 **安全防护** 页面设置全局默认设备数上限，适用于所有用户。

### 用户级覆盖

在 **用户管理** 中可以为每个用户单独设置设备上限：

| 设置值 | 效果 |
|--------|------|
| `> 0`（如 1/2/3） | 该用户最多同时 N 个设备在线 |
| `0` | 该用户不限制设备数 |

### 管理员豁免

管理员账号**不受**设备流控约束，可在任意数量的设备上使用。

## 活跃设备管理

管理员可在 **活跃设备** 页面：

- 查看各 Emby 实例下所有用户的在线设备
- 查看设备详情：客户端类型、IP 地址、最后活跃时间
- **手动踢出**指定设备

## 预缓存联动

被设备流控拦截的用户**不会触发预缓存**，避免在无法播放的情况下浪费 115 接口调用额度。

## 新用户默认值

新用户创建或从 Emby 同步时，自动继承全局默认设备上限设置。
