# 监控转发

对应前端侧边栏 **监控 → 监控转发**，基于 MTProto Session 实时监听 Telegram 频道消息，并按规则自动转发到目标频道。

## 页面结构

顶部标题栏 + 「添加规则」按钮。下方以卡片列表展示所有监控规则，每张卡片显示：

- **规则名称** + 启用状态标签（运行中 / 已停用）
- **源频道** — 被监听的频道名称列表
- **目标频道** — 转发目标频道名称列表
- **HDHive** 标签（如开启了自动解锁）
- 操作按钮：手动转发 / 编辑 / 删除

## 编辑弹窗配置项

点击「添加规则」或「编辑」按钮，弹出配置弹窗：

### 基本信息

| 配置项 | 说明 |
|--------|------|
| **规则名称** | 自定义名称 |
| **启用** | 开关，控制该规则是否生效 |
| **源频道** | 要监听的 Telegram 频道，支持搜索 TG 用户名或昵称，可添加多个 |
| **目标频道** | 转发消息的目标频道，同样支持搜索，可添加多个 |

### 关键词过滤

| 配置项 | 说明 |
|--------|------|
| **白名单** | 只转发包含这些关键词的消息。输入后回车添加，以药片标签展示 |
| **黑名单** | 过滤掉包含这些关键词的消息。同样输入后回车添加 |

::: tip 过滤优先级
白名单优先于黑名单。同时设置时，消息必须匹配白名单且不匹配黑名单才会被转发。
:::

::: details 消息模板
| 配置项 | 说明 |
|--------|------|
| **模板内容** | 自定义转发消息格式。每行一条规则：`查找文本 => 替换文本`（替换），或只写关键词（删除该行） |
:::

### HDHive 自动解锁

| 配置项 | 说明 |
|--------|------|
| **HDHive 自动解锁** | 开关。监控到 HDHive 链接时自动解锁 |
| **积分上限** | 单次解锁的最大积分消耗，超过则跳过。填 0 = 不限制 |

::: important 启用后，监控到频道中的 HDHive 链接会自动解锁并转存，消耗 HDHive 积分。建议配合积分上限使用。
:::

::: details 转发消息增强
| 配置项 | 说明 |
|--------|------|
| **显示来源** | 在转发消息中附带源频道信息 |
| **显示 Bot 搜索** | 在消息下方附加 Bot 搜索按钮 |
| **自定义搜索 URL** | 自定义搜索链接地址（留空使用默认） |
| **显示投稿按钮** | 在消息下方附加投稿按钮 |
| **自定义投稿 URL** | 自定义投稿链接地址 |
| **广告位 1 / 2** | 可配置两条自定义广告按钮（文本 + URL） |
:::

## 手动转发

卡片上的「手动转发」按钮打开弹窗，支持两种模式：

- **按消息 ID** — 输入具体消息 ID 进行手动转发
- **按时间范围** — 选择时间段，转发该时间段内的所有消息

执行后显示转发结果（成功/失败数量）。
