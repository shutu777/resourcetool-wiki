# 115 签到

对应前端侧边栏 **插件库 → 115 签到**，为每个 115 网盘账号独立配置每日自动签到计划，自动领取签到奖励。

## 功能概述

::: important 核心用途
115 网盘每日签到可获取积分和空间奖励。本插件支持为每个 115 账号分别配置签到时间，系统会在指定时间自动执行签到，无需手动操作。
:::

## 页面结构

进入插件后，页面以列表形式展示所有 115 账号的签到配置：

| 字段 | 说明 |
|------|------|
| **配置名称** | 115 账号的配置标识名 |
| **签到状态** | <Badge type="tip" text="已签到" /> 或 <Badge type="warning" text="未签到" /> |
| **签到时间** | 配置的每日自动签到时间 |
| **上次签到** | 最近一次成功签到的时间 |
| **操作** | 手动签到 / 编辑配置 |

## 配置项

| 配置项 | 说明 |
|--------|------|
| **启用自动签到** | 开关，控制该账号是否参与自动签到 |
| **签到时间** | 每日自动签到的执行时间（如 `08:00`） |

::: tip 随机延迟
系统会在设定时间的基础上自动添加随机延迟（0~5 分钟），避免所有账号同时请求 115 API 触发风控。
:::

## 使用方法

::::: steps

1. **进入插件**

   在前端侧边栏点击 **插件库 → 115 签到**

2. **配置签到**

   为每个 115 账号开启自动签到并设置签到时间

3. **查看结果**

   签到执行后可在列表中查看状态，详细日志可在 [系统配置](system-config.md) 的日志中查看

4. **手动签到**

   点击账号右侧「手动签到」按钮可立即执行一次签到

:::::

## 注意事项

::: warning Cookie 有效性
签到依赖有效的 115 Cookie。如果签到持续失败，请先通过 [115 健康检查](plugin-115-health.md) 确认 Cookie 状态。
:::
