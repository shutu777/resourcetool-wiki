# 同播复制

同播复制（EnableSync）功能允许多个用户同时播放同一文件，通过自动复制文件避免 115 单文件并发限制。

## 工作原理

115 网盘对同一文件的并发下载有限制。当多个用户同时播放同一视频时，同播复制会自动为后续用户创建文件副本，使每个用户都能获得独立的直链。

## 触发条件

```
用户 A 正在播放文件 X（pickcode 活跃中）
  → 用户 B 请求播放同一文件 X
  → 检测到不同用户播放同一 pickcode
  → 触发同播复制
```

## 核心特性

### 自动检测

系统通过 `activePickcodes` 缓存跟踪当前活跃的 pickcode：

- 用户播放时记录 pickcode + 用户标识
- 新请求到达时检查是否有**不同用户**正在播放同一文件
- 同一用户的重复请求不会触发同播复制

### 自动清理

副本文件在缓存过期后自动删除：

- 清理时间 = URL 缓存 TTL + 30 秒（留出缓冲）
- 删除副本文件
- 清空回收站（如配置了回收站密码）

### 三条路径覆盖

同播复制检测覆盖所有播放路径：

| 路径 | 检测时机 |
|------|---------|
| **缓存命中** | 从缓存条目中读取 pickcode |
| **CD2 快速路径** | 从 CD2 结果中获取 pickcode |
| **115 API 路径** | 从 API 结果中获取 pickcode |

## 配置

在 Emby 302 配置中启用 `EnableSync` 选项即可开启同播复制功能。

::: tip 使用建议
- 适合多用户共享同一 115 账号的场景
- 如果用户都是自备 Cookie，通常不需要启用此功能
- 开启后会增加少量 115 API 调用（复制 + 删除）
:::
