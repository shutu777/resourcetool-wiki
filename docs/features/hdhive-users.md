# HDHive 用户

对应前端侧边栏 **HDHive → HDHive 用户**，管理 HDHive Open API 用户，实现多用户负载均衡和额度监控。

## 页面结构

顶部标题栏 + 「添加用户」按钮 + 「刷新额度」按钮。下方以卡片网格展示所有 HDHive 用户。

## 用户卡片

每张卡片显示：

| 字段 | 说明 |
|------|------|
| **用户名** | HDHive 用户名（从 API 自动获取） |
| **剩余积分** | 当前可用积分数 |
| **今日消耗** | 当天已使用的积分数 |
| **API Key** | 脱敏显示的 API Key |
| **状态** | 正常（绿色）/ 额度不足（琥珀色）/ 异常（红色） |

操作按钮：编辑 / 删除。

## 添加/编辑用户弹窗

| 配置项 | 说明 |
|--------|------|
| **API Key** | HDHive Open API 的 API Key |

保存后系统自动调用 HDHive API 获取用户信息和额度。

## 刷新额度

点击顶部「刷新额度」按钮，一次性刷新所有用户的积分余额和消耗数据。系统也会定时自动刷新。

## 负载均衡

::: info 智能调度策略
配置多个 HDHive 用户后，系统按**剩余积分**智能调度：优先选择积分最充足的用户，单个用户积分耗尽时自动切换到下一个可用用户。
:::
