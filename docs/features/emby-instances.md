# 实例管理

对应前端侧边栏 **开服 → 实例管理**，集中管理每个 Emby 实例的 Bot 配置、运营模式、签到积分、同步导入默认值、媒体库屏蔽和求片系统。

::: important 为什么需要实例管理
在多实例部署场景中（如一台 Emby 做国产剧、另一台做欧美剧），每个实例需要独立的 Bot、运营策略和媒体库屏蔽规则。实例管理将这些配置从原来的 302 配置中拆分出来，以可折叠卡片的形式集中管理，避免来回切换页面。
:::

## 页面结构

页面以**可折叠卡片**形式展示所有 Emby 实例。每张卡片头部显示实例名称、Bot 运行状态、运营模式，点击头部可展开/收起。

::: tip 实例来源
实例列表来自 [Emby 助手](emby302.md) 中创建的 302 配置。如果页面提示「暂无实例」，请先在 Emby 助手中创建实例。
:::

## Bot 配置

每个实例拥有独立的 Telegram Bot，用于用户交互、通知推送和管理操作。

| 配置项 | 说明 |
|--------|------|
| **Bot Token** | 从 [@BotFather](https://t.me/BotFather) 获取的该实例专属 Token |
| **管理员 TG 账号** | 该实例的 Bot 管理员，接收通知并可使用管理员命令 |
| **TG 群组 ID** | 关联的 TG 群组，用于退群检测等功能，留空不启用 |
| **自定义欢迎消息** | 支持 `{name}` 占位符，留空使用默认欢迎语 |
| **封面图** | Bot 发送消息时的封面图片，支持 jpg/png/gif/webp |

### 配置步骤

::::: steps

1. **创建 Bot**

   在 Telegram 中找到 [@BotFather](https://t.me/BotFather)，发送 `/newbot` 创建 Bot，获取 Token

2. **填写 Token**

   将 Token 粘贴到 Bot Token 输入框，点击右侧「测试」按钮确认连接正常

3. **配置管理员**

   在「管理员 TG 账号」搜索框中搜索并添加管理员的 TG 用户名或昵称

4. **关联群组（可选）**

   如需启用退群检测，在「TG 群组 ID」中搜索并选择关联的 TG 群组

5. **保存并启动**

   点击卡片底部「保存」按钮，然后点击卡片头部的「启动 Bot」按钮

:::::

::: tip Bot 启停
每张卡片头部有「启动 Bot / 停止 Bot」按钮，可独立控制每个实例的 Bot 运行状态，无需重启整个服务。
:::

## 运营模式与签到积分

每个实例可独立选择运营模式，签到模式下用户通过每日签到获得积分并兑换各类权益。

### 运营模式

| 模式 | 说明 |
|------|------|
| **标准模式** | 管理员手动管理用户到期、续期等操作 |
| **签到模式** | 用户可通过 `/checkin` 每日签到获取积分，用积分兑换白名单、续期、解封、邀请码等权益 |

### 签到积分配置（仅签到模式）

选择签到模式后，展开以下配置项：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **启用签到** | 关 | 开启后用户可使用 `/checkin` 命令每日签到 |
| **签到最少积分** | 1 | 每次签到获取的最少积分 |
| **签到最多积分** | 10 | 每次签到获取的最多积分（实际值在最少~最多之间随机） |
| **白名单兑换积分** | 1000 | 用户使用 `/exchange` 兑换白名单身份所需积分 |
| **解封兑换积分** | 500 | 被封禁用户兑换解封所需积分 |
| **邀请码兑换积分** | 100 | 用户生成一个邀请码所需积分 |
| **兑换续期积分** | 300 | 用户主动兑换续期所需积分，填 0 关闭此功能 |
| **兑换续期天数** | 30 | 每次兑换续期增加的天数 |

### 积分自动续费

::: important 自动续费逻辑
开启后，到期扫描任务在检测到用户到期时，会优先检查用户积分余额。如果积分充足，自动扣除积分并续期，而不是直接禁用用户。续费成功后自动通知用户。
:::

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **启用自动续费** | 关 | 开关 |
| **自动续费积分** | 300 | 每次自动续费扣除的积分 |
| **自动续费天数** | 30 | 每次自动续费增加的天数 |

### 活跃度检测

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **启用活跃度检测** | 关 | 开启后管理员可使用 `/low_activity` 命令检测不活跃用户 |
| **不活跃天数** | 30 | 超过此天数未播放的用户视为不活跃 |

### 典型配置场景

::: details 小型私服推荐配置
- 运营模式：**签到模式**
- 签到积分：1 ~ 10
- 白名单兑换：1000 积分（约 100 天签到）
- 兑换续期：300 积分 / 30 天
- 自动续费：**开启**，300 积分 / 30 天

这样用户每天签到积累积分，到期时如果有足够积分会自动续费，不需要管理员手动操作。
:::

## 同步导入默认值

控制从 Emby 导入用户或定时同步新增用户时，默认设置的到期时间和白名单状态。

| 配置项 | 说明 |
|--------|------|
| **默认时长** | 永久 / 天卡(1天) / 周卡(7天) / 月卡(30天) / 年卡(365天) |
| **白名单** | 仅在默认时长为「永久」时可勾选；勾选后新导入用户直接为白名单身份 |

::: tip 有到期时间时不可设白名单，因为白名单用户不受到期扫描影响，二者互斥。
:::

## 媒体库屏蔽

管理员统一配置需要管控的媒体库，非白名单用户将被自动隐藏这些媒体库。

::: important v1.6.5 变更
媒体库屏蔽从之前的用户自助操作改为管理员统一配置。用户面板不再提供自助入口，由管理员在实例管理中统一设置屏蔽规则和白名单用户。
:::

| 配置项 | 说明 |
|--------|------|
| **启用媒体库屏蔽** | 总开关 |
| **受管控的媒体库** | 勾选需要管控的媒体库名称，支持从 Emby 自动拉取或手动输入 |
| **白名单用户** | 白名单用户可访问受管控媒体库，Emby 管理员自动跳过 |

### 配置步骤

::::: steps

1. **开启屏蔽**

   打开「启用媒体库屏蔽」开关

2. **选择受管控媒体库**

   点击「从 Emby 拉取媒体库」按钮获取媒体库列表，勾选需要管控的媒体库。也可手动输入媒体库名称

3. **配置白名单用户**

   添加允许访问受管控媒体库的白名单用户。Emby 管理员账号自动跳过屏蔽，无需手动添加

4. **保存配置**

   点击保存后，系统自动同步所有用户的媒体库权限。新建/导入用户也会自动应用屏蔽规则

:::::

::: warning 配置变更后自动全量同步
保存配置时系统会自动同步所有用户的媒体库权限，如果用户数量较多可能需要几分钟。
:::

## 求片系统

允许用户通过 Bot 或用户面板搜索 TMDB 并提交求片请求，管理员可在后台处理。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **启用求片** | 关 | 总开关 |
| **电影求片限制** | 每 7 天最多 3 次 | 限制周期内每用户最多提交的电影求片次数，填 0 不限制 |
| **剧集求片限制** | 每 7 天最多 3 次 | 限制周期内每用户最多提交的剧集求片次数，填 0 不限制 |

::: tip 求片管理
开启求片后，管理后台会出现「求片管理」页面，详见 [求片系统](media-requests.md)。
:::
