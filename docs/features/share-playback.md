# 分享播放

分享播放是 Emby 302 直链服务的扩展功能，允许通过 115 分享链接实现跨账号播放。

## 工作原理

当用户播放的文件存在于其他 115 账号时，通过分享机制获取直链播放，无需将文件转存到用户自己的网盘。

## 三级兜底机制

分享播放采用三级兜底策略，确保最大化播放成功率：

```
分享秒传 → 分享转存 → 分享直链
```

### 1. 分享秒传

通过 SHA1 秒传到目标账号，速度最快：

- 直接将文件秒传到目标 115 账号
- 获取目标账号的直链播放
- 支持 sign_check 校验

### 2. 分享转存

秒传失败时，通过分享接收转存：

- 调用 `share/receive` API 将文件转存到目标账号
- 获取转存文件的直链
- 转存文件缓存过期后自动清理，避免占用空间

### 3. 分享直链

直接通过分享下载 API 获取直链：

- 副号直接调用分享下载 API
- 无需秒传或转存操作
- 减少一次 API 调用

## 多副号轮询

配置多个分享副号时：

- **随机选择** — 自动随机选择一个副号
- **失败重试** — 单个失败自动尝试下一个
- **提升可用性** — 多副号分摊请求压力

## 分享文件清理

通过分享下载 API 接收的文件会在**缓存过期后自动删除**，避免副号空间占用持续增长。

## 秒传回退优化

当秒传全部失败时，已获取的分享直链会传递给回退逻辑复用，避免重复调用 `GetShareDownloadURL`。

## 自备模式支持

自备 Cookie 用户同样享有完整的分享播放能力：

```
分享秒传 → 分享转存 → 分享直链（三级兜底）
```

## 日志追踪

分享播放提供详细的日志记录：

| 日志类型 | 内容 |
|---------|------|
| 分享解析开始 | share / fileId / source / target / ua |
| 目标覆盖/注入 | share_map 与用户分流命中情况 |
| 秒传目标 | `115-1 → 115-2` 格式 |
| 转存状态 | 开始 / 完成 / 已存在 |
| 直链就绪 | 最终获取到的直链信息 |
