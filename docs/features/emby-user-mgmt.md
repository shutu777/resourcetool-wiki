# Emby 用户

对应前端侧边栏 **开服 → Emby 用户**，是 Emby 开服的核心管理页面，管理所有 Emby 实例下的用户。

## 页面结构

顶部标题栏包含以下操作按钮：

:::: card-grid
::: card title="创建用户" icon="material-symbols:person-add"
手动创建新 Emby 用户，设置实例、密码、网盘模式和到期时间
:::
::: card title="从 Emby 导入" icon="material-symbols:download"
从 Emby Server 批量导入已有用户到 RT 数据库
:::
::: card title="同步到 Emby" icon="material-symbols:sync"
将 RT 用户批量同步到 Emby Server（创建/更新状态）
:::
::::

下方为搜索栏（按用户名搜索）+ Emby 实例筛选下拉 + 状态筛选下拉 + 网盘模式筛选下拉。

### 用户列表表格

| 列名 | 说明 |
|------|------|
| **用户名** | Emby 用户名 |
| **Emby 实例** | 所属 Emby 302 实例名称 |
| **网盘模式** | 「共享」或「自备」标签 |
| **状态** | active（绿色）/ disabled（红色）/ expired（琥珀色） |
| **到期时间** | 到期日期，永久用户显示「永久」标签 |
| **TG 绑定** | 已绑定的 Telegram 用户名或 ID |
| **设备数** | 当前设备数 / 设备上限 |
| **操作** | 编辑 / 同步到 Emby / 删除 |

## 创建用户弹窗

| 配置项 | 说明 |
|--------|------|
| **Emby 实例** | 选择用户所属的 Emby 302 实例 |
| **用户名** | Emby 用户名 |
| **密码** | Emby 密码 |
| **网盘模式** | 共享 / 自备 |
| **到期时间** | 日期选择器，或勾选「永久」 |

::: details 编辑用户弹窗
除创建时的字段外，还可编辑：

| 配置项 | 说明 |
|--------|------|
| **状态** | active / disabled |
| **设备上限** | 最大同时播放设备数 |
| **TG 用户 ID** | 绑定的 Telegram 用户 ID |
| **115 配置** | 自备模式下绑定的 115 账号配置 |
| **续期** | 快捷续期按钮（+7天 / +30天 / +90天 / +365天 / 永久） |
:::

## 从 Emby 导入

::: info 导入逻辑
点击「从 Emby 导入」按钮后弹出弹窗，选择 Emby 实例，系统自动拉取 Emby Server 上的用户列表，对比 RT 数据库：

| 情况 | 处理 |
|------|------|
| 用户不存在 | 自动创建记录，默认 shared 模式 |
| 同名同实例 | 跳过，标记为「已存在」 |

导入完成后显示导入结果汇总。
:::

## 同步到 Emby

::: info 同步策略
- **新用户** — 在 Emby 上创建账号并设置权限
- **已有用户** — 仅同步启用/禁用状态，不覆盖已有权限
:::
