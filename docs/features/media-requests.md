# 求片系统

对应前端侧边栏 **开服 → 求片管理**，管理用户通过 Bot 或用户面板提交的影视求片请求。

::: important 前置条件
求片系统需要先在 [实例管理](emby-instances.md) 中开启「启用求片」开关，并在 Emby 302 配置中设置 TMDB API Key 才能正常使用。
:::

## 工作流程

整个求片过程分为用户提交和管理员处理两个阶段：

```
用户搜索 TMDB → 确认影片 → 填写备注 → 提交求片
                                              ↓
管理员查看 → 完成/拒绝 → 用户收到通知
```

### 用户端（Bot / 用户面板）

用户可以通过两种方式提交求片：

**通过 Bot：**

::::: steps

1. **发起搜索**

   在 Bot 中使用菜单进入求片功能，输入想要搜索的影片名称

2. **选择影片**

   Bot 返回 TMDB 搜索结果列表，包含海报、年份、类型等信息，点击选择目标影片

3. **选择求片类型**

   - **新片求片** — 媒体库中没有的影片
   - **缺集报告** — 剧集缺少某些集数
   - **异常报告** — 影片存在画质、字幕等问题

4. **填写备注（可选）**

   补充说明需求细节，如「缺少第 5 集」「字幕不同步」等

5. **提交**

   确认后提交求片请求，系统通知管理员

:::::

**通过用户面板：**

用户面板中的「我的求片」页面提供相同的搜索和提交功能，以卡片形式展示历史求片记录和当前状态。

### 求片频率限制

为防止滥用，每个用户在限制周期内的求片次数受到限制。默认每 7 天最多 3 次电影求片和 3 次剧集求片，管理员可在 [实例管理](emby-instances.md) 的求片系统配置中调整。

---

## 管理员端（求片管理页面）

### 页面结构

页面顶部显示统计卡片（总求片数、待处理、已完成、已拒绝），下方为筛选栏和求片列表。

### 筛选与搜索

| 筛选项 | 说明 |
|--------|------|
| **实例选择** | 多实例时可按实例筛选 |
| **片名搜索** | 按影片名称关键词搜索 |
| **用户名搜索** | 按提交用户的用户名搜索 |
| **状态筛选** | 全部 / 待处理 / 已完成 / 已拒绝 |
| **类型筛选** | 全部 / 新片求片 / 缺集报告 / 异常报告 |

### 求片列表

每条记录显示以下信息：

| 字段 | 说明 |
|------|------|
| **影片信息** | TMDB 海报缩略图、中英文片名、年份、类型（电影/剧集） |
| **用户** | 提交求片的 Emby 用户名 |
| **求片类型** | 新片求片 / 缺集报告 / 异常报告 |
| **备注** | 用户填写的补充说明 |
| **状态** | <Badge type="warning" text="待处理" /> / <Badge type="tip" text="已完成" /> / <Badge type="danger" text="已拒绝" /> |
| **时间** | 提交时间 |

### 处理求片

::::: steps

1. **查看待处理请求**

   将状态筛选设为「待处理」，查看所有等待处理的求片

2. **完成求片**

   资源已入库后，点击「完成」按钮，系统自动通知用户求片已完成

3. **拒绝求片**

   无法满足的请求，点击「拒绝」按钮，系统自动通知用户

4. **删除记录**

   已完成或已拒绝的记录可点击「删除」清理

:::::

::: tip 移动端适配
移动端自动切换为卡片视图，操作按钮显示在卡片底部，桌面端鼠标悬停在行上显示操作按钮。
:::
