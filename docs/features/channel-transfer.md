# 监控转存

对应前端侧边栏 **监控 → 监控转存**，监控指定 Telegram 频道中的链接并自动转存到 115 网盘。

## 页面结构

顶部标题栏 + 「添加规则」按钮。下方以卡片列表展示所有转存规则，每张卡片显示：

- **规则名称** + 启用状态标签（运行中 / 已停用）
- **源频道** — 被监听的频道名称列表
- **绑定的 115 账号** — 每个 115 配置名显示为一个药片标签
- 操作按钮：编辑 / 删除

## 编辑弹窗配置项

点击「添加规则」或「编辑」按钮，弹出配置弹窗：

### 基本信息

| 配置项 | 说明 |
|--------|------|
| **规则名称** | 自定义名称 |
| **启用** | 开关，控制该规则是否生效 |
| **源频道** | 要监听的 Telegram 频道，支持搜索 TG 用户名或昵称，可添加多个 |

### 115 转存目标

每条规则可绑定 **多个** 115 账号（多目标转存），点击「添加 115 目标」按钮逐一配置：

| 配置项 | 说明 |
|--------|------|
| **115 配置** | 从已有 115 账号下拉选择 |
| **转存目录** | 115 链接的转存目标目录，通过目录选择器浏览 115 网盘选择 |
| **离线目录** | magnet/ed2k 的离线下载目录，同样使用目录选择器 |

每个 115 目标可独立配置转存目录和离线目录，不同 115 账号的资源可以转存到不同目录，便于分类管理。

### 关键词过滤

| 配置项 | 说明 |
|--------|------|
| **白名单** | 只转存包含这些关键词的消息中的链接。输入后回车添加，以药片标签展示 |
| **黑名单** | 过滤掉包含这些关键词的消息。同样输入后回车添加 |

::: tip 过滤优先级
白名单优先于黑名单。同时设置时，消息必须匹配白名单且不匹配黑名单才会被转存。
:::

::: important 多目标转存
每条规则可绑定多个 115 账号，同一条消息中的链接会同时转存到所有已配置的 115 目标，每个目标可独立配置转存和离线目录。
:::

## 目录选择器

::: tip 目录选择器
与 [115 助手](pan115-config.md) 中相同的目录选择器，带面包屑导航的 115 网盘目录浏览弹窗。
:::
