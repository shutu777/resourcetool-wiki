# 插件库

ResourceTool 内置多个自动化插件，覆盖网盘签到、健康检查、Telegram 自动化等场景。所有插件均在前端侧边栏 **插件库** 中统一管理。

---

## 115 健康检查

批量检测所有 115 网盘账号的 Cookie 有效性，快速定位失效配置。

::: important 为什么需要健康检查
115 网盘 Cookie 会定期过期，过期后转存、离线、302 播放等功能均会失效。本插件可一键检测所有配置的 Cookie 状态，避免因 Cookie 失效导致服务中断。
:::

### 使用方法

::::: steps

1. **进入插件**

   侧边栏点击 **插件库 → 115 健康检查**

2. **执行检测**

   点击页面顶部「一键检测」按钮，系统逐个调用 115 API 验证每个账号的 Cookie

3. **查看结果**

   - <Badge type="tip" text="正常" /> — Cookie 有效，无需操作
   - <Badge type="danger" text="失效" /> — Cookie 已过期，需前往 [115 助手](pan115-config.md) 更新

:::::

::: tip 建议每周执行一次健康检查，或在发现转存/播放异常时优先检查。频繁调用 115 API 可能触发风控，不要短时间内反复执行。
:::

## 115 签到

为每个 115 网盘账号配置每日自动签到计划，自动领取签到奖励（积分和空间）。

### 配置项

| 配置项 | 说明 |
|--------|------|
| **启用自动签到** | 开关，控制该账号是否参与自动签到 |
| **签到时间** | 每日自动签到的执行时间（如 `08:00`） |

::: tip 随机延迟
系统会在设定时间基础上自动添加 0~5 分钟随机延迟，避免所有账号同时请求 115 API 触发风控。
:::

### 使用方法

::::: steps

1. **进入插件**

   侧边栏点击 **插件库 → 115 签到**

2. **配置签到**

   为每个 115 账号开启自动签到并设置签到时间

3. **手动签到**

   点击账号右侧「手动签到」按钮可立即执行一次

4. **查看结果**

   签到后列表中显示 <Badge type="tip" text="已签到" /> 或 <Badge type="warning" text="未签到" />，详细日志可在系统日志中查看

:::::

::: warning Cookie 有效性
签到依赖有效的 115 Cookie。如果签到持续失败，请先通过上方的 115 健康检查确认 Cookie 状态。
:::

## CloudDrive2 助手

配置 CloudDrive2（CD2）的连接信息和挂载路径映射，为 Emby 302 的**路径替换**播放模式提供 gRPC 加速。

::: important 核心用途
在路径替换模式中，系统需要将 Emby 媒体路径转换为 115 网盘的文件 Pickcode。CD2 助手通过 gRPC 协议直连 CD2 服务，实现毫秒级路径查询，大幅提升播放响应速度。
:::

### 连接配置

| 配置项 | 说明 |
|--------|------|
| **CD2 地址** | CloudDrive2 服务地址（如 `http://192.168.1.100:19798`） |
| **用户名** | CD2 登录用户名 |
| **密码** | CD2 登录密码 |
| **测试连接** | 验证连接是否成功 |

### 挂载路径映射

配置 Emby 媒体库路径与 CD2 挂载路径之间的对应关系。支持添加多组映射。

| 配置项 | 说明 |
|--------|------|
| **Emby 路径前缀** | Emby 媒体库中的路径前缀（如 `/mnt/media/`） |
| **CD2 挂载路径** | 对应的 CD2 115 网盘挂载路径（如 `/115/media/`） |

::: details 路径映射示例
假设 Emby 媒体库路径为 `/mnt/media/movies/xxx.mkv`，CD2 将 115 网盘挂载到 `/115/media/`，则：

- **Emby 路径前缀** → `/mnt/media/`
- **CD2 挂载路径** → `/115/media/`

系统会将 `/mnt/media/movies/xxx.mkv` 转换为 `/115/media/movies/xxx.mkv`，然后通过 CD2 gRPC 查询对应的 115 Pickcode。
:::

### 使用方法

::::: steps

1. **安装 CloudDrive2**

   确保 CD2 已安装并挂载了 115 网盘

2. **配置连接**

   填写 CD2 的地址、用户名、密码，点击「测试连接」

3. **配置路径映射**

   添加 Emby 媒体路径与 CD2 挂载路径的对应关系

4. **在 Emby 助手中启用**

   前往 [Emby 助手](emby302.md) 配置路径替换模式，系统将自动使用 CD2 gRPC 查询

:::::

::: warning CD2 服务需保持运行。如果停止或挂载丢失，路径替换模式会回退到 Pickcode 模式。建议 CD2 与 RT 部署在同一内网。
:::

## HDHive 签到

通过 HDHive Open API 自动完成每日签到，领取积分奖励。HDHive 积分是解锁影视资源的货币。

### 签到模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **普通签到** | 稳定获取固定积分，无风险 | 积分充裕或保守型用户 |
| **赌狗模式** | 随机积分，上限更高但有概率获得较少 | 追求高收益的用户 |

::: caution 赌狗模式的积分收益随机波动，极端情况下可能不如普通签到，请根据积分储备谨慎选择。
:::

### 配置项

| 配置项 | 说明 |
|--------|------|
| **启用自动签到** | 开关 |
| **签到时间** | 每日自动签到执行时间 |
| **签到模式** | 普通签到 / 赌狗模式 |

### 使用方法

::::: steps

1. **配置 HDHive 用户**

   确保已在 [HDHive 助手](hdhive-users.md) 中添加了 HDHive API Key

2. **进入插件**

   侧边栏点击 **插件库 → HDHive 签到**

3. **设置签到策略**

   为每个 HDHive 用户选择签到模式和时间

4. **查看结果**

   签到后列表显示获得积分和账户余额

:::::

::: warning 签到依赖有效的 HDHive API Key。失败时请在 [HDHive 助手](hdhive-users.md) 中确认 API Key 状态。
:::

## Telegram 助手

配置 Telegram MTProto Session 和主 Bot，是频道监控、消息转发、用户交互等功能的基础。

::: important 核心用途
Telegram 助手是 RT 与 Telegram 通信的枢纽。MTProto Session 用于监听频道消息实现自动转发/转存；Bot Token 用于用户交互和通知推送。
:::

### MTProto Session

用于登录 Telegram 用户账号，实现频道消息的实时监听。

| 配置项 | 说明 |
|--------|------|
| **API ID** | 从 [my.telegram.org](https://my.telegram.org) 获取 |
| **API Hash** | 从 [my.telegram.org](https://my.telegram.org) 获取 |
| **手机号** | Telegram 账号的手机号码 |

::: caution Session 安全
Session 等同于 Telegram 账号的登录凭证，请妥善保管。泄露可能导致账号被他人控制。
:::

**登录流程：**

::::: steps

1. **填写 API 信息** — 输入 API ID、API Hash 和手机号

2. **获取验证码** — 点击「发送验证码」，Telegram 会发送验证码

3. **输入验证码** — 在弹窗中输入收到的验证码

4. **二步验证（如有）** — 如果账号开启了二步验证，还需输入密码

5. **登录成功** — Session 状态变为 <Badge type="tip" text="已登录" />

:::::

### 主 Bot 配置

| 配置项 | 说明 |
|--------|------|
| **Bot Token** | 从 [@BotFather](https://t.me/BotFather) 获取 |
| **白名单** | 允许使用 Bot 的用户 ID 列表，留空不限制 |
| **Chat ID** | Bot 发送通知的目标聊天 ID |

::: info Bot 与 Session 的区别
- **MTProto Session** — 监听频道消息（转发/转存的前提）
- **Bot Token** — 用户交互和通知推送（搜索/管理的前提）

两者独立工作，可根据需要单独配置。
:::

## Telegram 签到

通过 MTProto Session 定时向指定群组/频道/Bot 发送签到指令，自动完成各类 TG 签到任务。

### 添加任务

| 配置项 | 说明 |
|--------|------|
| **任务名称** | 自定义名称，便于识别 |
| **启用** | 开关 |
| **目标 Chat ID** | 签到目标的聊天 ID |
| **签到指令** | 要发送的签到消息（如 `/sign`、`签到`、`打卡`） |
| **签到时间** | 每日自动执行时间 |
| **Cron 表达式** | 高级用户可自定义执行频率 |

::: tip 签到指令支持任意文本，不限于 Bot 命令。可以配置为目标群组要求的任何格式。
:::

### 使用方法

::::: steps

1. **前置条件** — 确保已在上方 Telegram 助手中配置 MTProto Session 并成功登录

2. **添加签到任务** — 点击「添加任务」，填写目标 Chat ID 和签到指令

3. **设置签到时间** — 配置每日执行时间或 Cron 表达式

4. **查看结果** — 签到后任务卡片显示状态和最近签到时间

:::::

::: warning Telegram 签到通过 MTProto Session 发送消息（非 Bot），请确保 Session 正常。请遵守目标群组规则，避免因自动化操作被封禁。
:::

::: details 获取 Chat ID
- 将 Bot 拉入群组，使用 `/id` 命令获取
- 使用 [@userinfobot](https://t.me/userinfobot) 转发消息获取
- 频道 ID 通常为负数（如 `-1001234567890`）
:::
