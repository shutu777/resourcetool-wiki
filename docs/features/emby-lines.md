# 访问线路

对应前端侧边栏 **开服 → 访问线路**，为每个 Emby 实例配置多条访问线路，用户可在 Bot 中切换线路。

## 页面结构

顶部标题栏 + Emby 实例选择下拉。选择实例后，下方显示该实例的线路列表和「添加线路」按钮。

## 线路列表

线路以卡片列表展示，支持 **拖拽排序**（按住左侧手柄上下拖动调整顺序）。每条线路显示：

- **线路名称** — 如「深圳电信」「香港 CN2」
- **线路 URL** — 完整的 Emby 访问地址
- **隐藏标签** — 是否为隐藏线路
- **永久可见标签** — 隐藏线路是否对永久用户可见

## 添加/编辑线路弹窗

| 配置项 | 说明 |
|--------|------|
| **线路名称** | 自定义名称 |
| **线路 URL** | Emby 访问地址（如 `https://emby.example.com:443`） |
| **隐藏** | 开关。开启后普通用户不可见 |
| **永久可见** | 开关。开启后即使隐藏，永久用户仍可见 |

## 探测功能

每条线路右侧有「探测」按钮，点击后系统自动测试该线路的可用性和延迟：

- <Badge type="tip" text="在线" /> 显示延迟毫秒数
- <Badge type="danger" text="离线" /> 线路不可达

顶部还有「全部探测」按钮，一次性测试所有线路。

## 排序

拖拽调整线路顺序后自动保存。线路顺序决定了用户在 Bot 中看到的线路排列。
