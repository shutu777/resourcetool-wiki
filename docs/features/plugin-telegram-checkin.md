# Telegram 签到

对应前端侧边栏 **插件库 → Telegram 签到**，通过 Telegram Bot 指令触发各类定时签到与自动化任务。

## 功能概述

::: important 核心用途
部分 Telegram 群组或频道提供签到机制（如积分、经验等）。本插件允许配置定时签到任务，由系统通过 Bot 自动发送签到指令，省去手动操作。
:::

## 页面结构

进入插件后，页面以卡片列表展示所有签到任务，每张卡片显示：

| 字段 | 说明 |
|------|------|
| **任务名称** | 自定义的签到任务标识 |
| **目标** | 签到的目标群组/频道/Bot |
| **签到状态** | <Badge type="tip" text="已签到" /> 或 <Badge type="warning" text="未签到" /> |
| **签到时间** | 配置的每日签到时间 |
| **上次签到** | 最近一次执行签到的时间 |
| **操作** | 手动签到 / 编辑 / 删除 |

## 添加/编辑任务

| 配置项 | 说明 |
|--------|------|
| **任务名称** | 自定义名称，便于识别 |
| **启用** | 开关，控制任务是否自动执行 |
| **目标 Chat ID** | 签到目标的聊天 ID（群组/频道/Bot） |
| **签到指令** | 要发送的签到消息或命令（如 `/sign`、`签到` 等） |
| **签到时间** | 每日自动执行的时间 |
| **Cron 表达式** | 高级用户可使用 Cron 表达式自定义执行频率 |

::: tip 灵活的签到指令
签到指令支持任意文本，不限于 Bot 命令。可以配置为 `/sign`、`签到`、`打卡` 等目标群组要求的任意格式。
:::

## 使用方法

::::: steps

1. **前置条件**

   确保已在 [Telegram 助手](plugin-telegram-config.md) 中配置了 MTProto Session 并成功登录

2. **添加签到任务**

   点击「添加任务」，填写任务名称、目标 Chat ID 和签到指令

3. **设置签到时间**

   配置每日自动签到的时间，或使用 Cron 表达式

4. **查看执行结果**

   签到执行后可在任务卡片中查看状态和最近签到时间

:::::

## 注意事项

::: warning MTProto Session 依赖
Telegram 签到通过 MTProto Session 发送消息，而非 Bot。请确保 Session 状态正常，否则签到将无法执行。
:::

::: caution 目标群组规则
请遵守目标群组/频道的规则，避免因自动化签到被管理员封禁。部分群组可能禁止自动化操作。
:::

::: details 获取 Chat ID
可以通过以下方式获取目标的 Chat ID：
- 将 Bot 拉入群组，使用 `/id` 命令获取
- 使用 [@userinfobot](https://t.me/userinfobot) 转发消息获取
- 频道 ID 通常为负数（如 `-1001234567890`）
:::
